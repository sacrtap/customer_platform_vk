# 客户运营中台 UI/UX 原型方案

## 交付文件

- `prototype/index.html`：单文件交互式高保真原型，可直接用浏览器打开。

## 推荐设计方向

**紧凑型企业数据驾驶舱**：面向内部运营、结算、分析人员，优先保证信息密度、数据可读性和任务处理效率，而不是营销型视觉表现。

### 设计原则

1. **核心指标先行**：每个关键页面首屏展示 4-6 个核心 KPI，快速回答“是否正常、哪里异常、下一步做什么”。
2. **筛选与结果同屏**：客户、分析、结算页面统一采用顶部筛选条 + 指标条 + 图表/表格布局，减少跳转。
3. **异常优先排序**：余额不足、同步失败、结算失败、临期回款等风险信息使用清晰状态标签和行动按钮。
4. **少装饰、强层级**：浅灰背景、白色卡片、蓝色主操作、青/绿/黄/红表达趋势和风险。
5. **可追溯操作**：客户详情、结算、系统治理页面突出时间线、操作者、状态和下一步动作。

## 视觉系统

- **主色**：深蓝 `#1D4ED8`，用于主按钮、关键趋势线、当前导航。
- **辅助色**：青色 `#0891B2`，用于次级图表和同步状态。
- **成功/预警/危险**：`#059669` / `#D97706` / `#DC2626`。
- **背景**：`#F6F8FB`，降低密集表格的视觉噪声。
- **文字**：主文本 `#0F172A`，辅助文本 `#475569`，保证浅色模式对比度。
- **圆角/阴影**：中等圆角和轻阴影，只用于分组和强调，不使用强玻璃拟态。

## 信息架构建议

以现有侧边栏分组结构为主，原型页面实现清单作为对照：

- **核心功能**：仪表盘、客户管理、结算管理（余额管理、计费规则、结算单）、画像管理。
- **运营分析**：消耗分析、回款分析、健康度分析、画像分析、预测回款。
- **系统管理**：用户管理、角色管理、同步日志、审计日志、数据库管理。
- **落地优先级**：P0 首页/客户/消耗/结算；P1 画像/健康度/回款；P2 用户/角色/日志。

## 侧边栏二级功能菜单方案

### 结构

- **总览**：设计总览、运营工作台。
- **核心功能 / 客户管理**：客户列表、客户详情、标签与画像。
- **核心功能 / 结算管理**：余额管理、计费规则、结算单队列。
- **运营分析 / 客户分析**：消耗分析、回款分析、健康度分析、画像分析。
- **系统管理 / 系统治理**：用户管理、角色管理、同步日志、审计日志。

### 交互规则

- 一级菜单负责业务域分组，二级菜单负责具体任务入口，避免侧边栏只是一串平铺页面。
- 所有一级、父级、二级菜单均配备统一 SVG 图标；菜单文字统一包裹为 `.nav-text`，便于折叠态隐藏。
- 二级菜单右侧使用短标签提示任务属性，例如“筛选 / 360 / 风险 / 配置 / 趋势 / 预警”。
- 当前原型中的二级菜单复用已有关键页面作为落点；真实落地时再映射到具体路由。
- 移动端保持分组但允许纵向堆叠，避免压缩成不可读的多列小按钮。

### 侧边栏折叠/展开

- 侧边栏右侧提供悬浮圆形折叠按钮，点击可在完整导航与折叠导航之间切换；按钮使用蓝青渐变、固定单一箭头字形，并通过旋转表达方向。
- 展开态宽度约 `252px`，展示品牌、分组标题、一级菜单、二级菜单、任务短标签和说明卡片。
- 折叠态宽度约 `72px`，仅显示一级菜单图标；隐藏品牌文字、分组标题、二级菜单、提示标签、菜单文字和侧边栏说明文字。
- 折叠态会强制隐藏已展开的二级菜单，避免选中“客户列表”等二级入口后收起侧边栏时产生大块空白。
- 折叠态仍保留当前业务域高亮，例如选中“客户列表”后收起，会高亮父级“客户管理”。
- 折叠状态通过 `localStorage` 持久化，刷新页面后保持用户上次的展开/收起选择。
- 展开态父级菜单采用单展开逻辑；折叠态点击父级菜单直接进入对应业务域，不展开二级菜单。

## 覆盖的关键原型页面

1. **设计总览**：项目页面分组、设计目标、关键原型覆盖范围。
2. **运营工作台**：经营 KPI、趋势图、异常待办、优先跟进客户。
3. **客户管理**：客户指标、组合筛选、客户表格、健康度和余额风险。
4. **客户详情**：客户 360、画像、余额、消耗热力、结算与操作记录。
5. **消耗分析**：同步状态、趋势、设备分布、Top 客户排行。
6. **结算管理**：余额/应收/账单风险、生成结算单流程、账单队列。
7. **系统治理**：角色权限、高风险操作、系统健康、审计日志。

## 后续落地建议

- 第一阶段优先落地：`Home`、`customers/Index`、`customers/Detail`、`analytics/Consumption`、`billing` 相关页面。
- 保留现有 Vue 3 + Arco Design 组件体系，先通过布局、信息架构、CSS token 和图表配置统一体验，避免大规模替换组件库。
- 对表格页补齐：保存筛选、批量操作、异常优先排序、空状态说明。
- 对分析页补齐：同步质量提示、图表解释、点击联动明细、导出报告。
- 对结算页补齐：生成前预检查、失败原因结构化、金额影响排序。
