Recommended UI Direction

紧凑型企业数据驾驶舱

项目是企业内部客户运营中台，覆盖账号治理、客户信息、结算、画像、消耗/回款/健康度分析。推荐从“首页 + 客户 + 分析 + 结算 + 治理”五个高频入口建立统一信息架构。

Visual System

白底低噪声 + 蓝青强调

主色 #1D4ED8，辅助青色 #0891B2；浅灰背景承载密集信息，卡片仅用于分组，不做过度装饰。

Data UX

指标、趋势、异常三层表达

每页首屏固定：核心指标条、主要图表、异常队列；表格提供批量操作和保存筛选。

Interaction

少跳转、强筛选、可追溯

详情抽屉/侧栏承载预览；关键流程展示状态线、操作者、下一步动作，降低运营判断成本。

项目页面分组理解

以现有侧边栏分组为主，原型实现清单作为对照

核心功能

  • 仪表盘 / 运营工作台
  • 客户管理 / 客户详情
  • 结算管理：余额、计费规则、结算单
  • 画像管理：客户画像、等级、标签

运营分析

  • 消耗分析
  • 回款分析
  • 健康度分析
  • 画像分析 / 预测回款

系统管理

  • 用户管理
  • 角色管理
  • 同步日志
  • 审计日志 / 数据库管理

原型实现清单

  • 设计总览、运营工作台
  • 客户管理、客户详情
  • 消耗分析、结算管理
  • 系统治理

落地优先级

  • P0：首页、客户、消耗、结算
  • P1：画像、健康度、回款分析
  • P2：用户、角色、同步/审计日志
  • 原则：高频和高风险路径先改

关键原型覆盖

优先覆盖最高频、最高风险路径

运营工作台

把待办、预警、同步状态和经营指标压缩在一屏。

客户管理/详情

列表快速筛选，详情页聚合画像、余额、消耗、结算历史。

消耗分析/结算

突出趋势、排行、异常账单和生成流程。
