## 三、建议：仅从 TD-002 起步

1. **先拆 `frontend/src/views/customers/Detail.vue`（2332 行）**，不超过周边逻辑、无事务面、拆分后收益最大；
2. **不一次性拆这 5 个文件**：即便 TD-002 整体“贵”，分块迭代也可以降低风险；
3. **完成一项、抽离公共部件** 后，再进入下一个（下一次建议 customers/Index.vue -> Dashboard.vue）；
4. **不等后端**：前端是 UI 拆分，后端不动逻辑，可单独推进。

## 四、暂不建议开启的原因

- **TD-001** 后端：核心结算模块，拆分涉及数据库事务/锁/结算状态机，建议在需求排期宽松时再单独评估。
- **TD-004/006**：需要升级 Python/Sanic + PostgreSQL，一次性升级风险中等，且无即时收益；建议在年度维护窗口合并执行。
- **TD-008/009**：低优先级，已有部分基础设施。

## 五、决策建议

- **接受建议（最高置信度）**: 从 customers/Detail.vue 组件拆分启动，按 PR -> 评审 -> 扫描 1 周完成原型，再批量推进其余文件。
- **跳过本次**: 关停，等交付压力降低。
