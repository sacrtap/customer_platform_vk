# 原型页面重构与优化方案

> **创建日期**: 2026-07-13
> **目标**: 在 `prototype/index.html` 原型文件中，为所有 22 个页面提供功能重构和优化的具体方案，包含交互设计示意，以便后续直接用于所有页面的 Vue 前端重构。

---

## 一、项目核心能力总结

客户运营中台覆盖五大核心域：

| 核心域 | 能力概述 | 关键页面 |
|--------|----------|----------|
| **账号治理** | RBAC 权限模型 + 自定义角色，细粒度权限控制 | 用户管理、角色管理 |
| **客户信息管理** | 统一客户基础信息 + 画像数据，Excel 导入/导出 | 客户列表、客户详情、标签管理 |
| **结算管理** | 3 种计费模式（定价/阶梯/包年），余额管理（先赠后实），完整结算流程 | 余额管理、计费规则、结算单 |
| **画像管理** | 双等级体系（规模等级 + 消费等级），自定义标签，组合筛选 | 客户详情画像Tab、画像分析 |
| **客户分析** | 消耗分析、回款分析、健康度分析、画像分析四大维度，预测回款 | 5 个分析页 |

## 二、原型设计核心理念

来源：`prototype/design.md` + `prototype/index.html`

### 设计原则

1. **核心指标先行**：首屏 4-6 个 KPI 卡片，快速回答"是否正常、哪里异常、下一步做什么"
2. **筛选与结果同屏**：顶部筛选条 + 指标条 + 图表/表格，减少跳转
3. **异常优先排序**：余额不足、同步失败、结算失败、临期回款等风险使用状态标签 + 行动按钮
4. **少装饰、强层级**：浅灰背景、白色卡片、蓝色主操作、语义色表达趋势和风险
5. **可追溯操作**：客户详情、结算、系统治理页面突出时间线、操作者、状态和下一步动作

### 设计令牌

- 主色 `#1D4ED8`，辅助青色 `#0891B2`
- 中性色：`--bg: #F6F8FB`、`--panel: #FFFFFF`、`--ink: #0F172A`、`--muted: #475569`
- 圆角：卡片 `16px`，按钮/输入框 `12px`，标签 `999px`
- 阴影：`0 1px 2px rgba(0,0,0,.04)`（轻量卡片）
- 侧边栏深色渐变 + 顶栏毛玻璃

### 3.2 运营工作台（Home）⭐ P0

**现状**: 6 KPI 条 + Hero 区（趋势图 + 待办）+ 优先跟进客户表格
**优化方案**:

| 优化项 | 描述 | 交互设计示意 |
|--------|------|-------------|
| KPI 卡片可点击下钻 | 每个 KPI 卡片点击跳转到对应分析页 | `@click` → 路由跳转 + hover 光标变手型 |
| 添加快捷操作面板 | Hero 区右侧待办下方新增"快捷操作"卡片 | 4 宫格按钮：新建客户 / 生成结算单 / 数据同步 / 导出报告 |
| 异常自动排序开关 | 待办列表添加排序切换：按影响金额 / 按临期程度 | Toggle 切换 + 列表实时重排序 |
| 实时同步状态条 | 顶部添加同步状态指示条（脉冲动画） | 绿色圆点 + "同步正常" / 红色 + "同步异常" |
| 优先跟进客户增强 | 表格行添加"一键操作"下拉 | 行内下拉菜单：提醒充值 / 生成结算单 / 查看详情 |
| 趋势图指标切换 | 图表 Tab 添加更多维度：消耗 / 回款 / 客户数 / 健康度 | Tab 切换触发图表数据切换 |

**交互流程示意**:
```
首屏加载 → 6 KPI 淡入 → 趋势图渲染 → 待办列表加载
     ↓
用户点击 KPI "低余额客户" → 跳转客户列表（带筛选：余额<7天）
     ↓
用户点击待办项 → 展开详情 → 操作按钮（处理/分配/忽略）
     ↓
用户切换趋势 Tab → 图表平滑切换数据 → 保持时间轴不变
```

### 3.3 客户管理列表（Customers）⭐ P0

**现状**: 4 KPI + 筛选 + 客户表格（logo + 进度条 + 标签）
**优化方案**:

| 优化项 | 描述 | 交互设计示意 |
|--------|------|-------------|
| 保存筛选视图 | 筛选区添加"保存视图"按钮，支持命名保存 | 弹窗输入名称 → 下拉菜单快速切换已保存视图 |
| 列自定义 | 表头添加列设置图标 | 弹出面板勾选显示列 → 拖拽排序 → localStorage 持久化 |
| 异常优先排序 | 表格添加"异常优先"排序开关 | Toggle 开启 → 行按健康度/余额风险自动排序 |
| 卡片/表格视图切换 | 表格右上角添加视图切换按钮 | 表格视图 ↔ 卡片网格视图（每卡片含 logo/名称/KPI/标签）|
| 快速预览抽屉 | 表格行 hover 显示"预览"浮层 | Hover 0.5s → 右侧 Drawer 滑出客户摘要（余额/消耗/健康度/最近操作）|
| KPI 联动筛选 | KPI 卡片点击联动筛选表格 | 点击"高风险客户 58" → 表格自动筛选健康度=高风险 |
| 批量操作增强 | 批量工具栏添加更多操作 | 批量编辑 / 批量分配负责人 / 批量导出 / 批量设标签 |

**交互流程示意**:
```
进入客户列表 → KPI 加载 → 表格加载
     ↓
用户设置筛选条件 → 点击"保存视图" → 输入"高价值房产客户" → 保存
     ↓
下次进入 → 下拉选择"高价值房产客户" → 筛选条件自动恢复
     ↓
用户 hover 表格行 → 0.5s 后右侧 Drawer 滑出客户 360 预览
     ↓
用户点击"详情" → 跳转客户详情页
```

| 快捷操作栏 | 顶部操作区添加常用操作快捷按钮 | 充值 / 生成结算单 / 编辑画像 / 发送提醒 |
| 关联客户推荐 | 画像 Tab 底部添加"关联客户"区块 | 同行业/同规模客户卡片列表 |

**交互流程示意**:
```

     ↓
用户点击"充值" → 充值弹窗 → 充值成功 → KPI 刷新
     ↓

     ↓
用户点击时间线事件 → 展开详情 → 查看操作人和变更内容
```
