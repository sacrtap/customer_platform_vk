## 1. 背景与目标

### 1.1 问题描述

侧边栏"画像管理"菜单点击后跳转到 `/profiles`，但该路由未定义，导致页面空白。

### 1.2 目标

创建一个画像管理仪表盘页面，作为画像模块的总览入口，汇总展示画像统计数据、可视化图表和快捷操作。

### 2.3 功能模块

#### 2.3.1 统计卡片区域 (StatsCards)

4 个统计卡片，使用 Arco Design `a-statistic` 组件：

| 卡片名称     | 数据来源                                  | 展示逻辑                           |
| ------------ | ----------------------------------------- | ---------------------------------- |
| 客户总数     | `getScaleStats()` 中各等级数量求和        | 数字显示                           |
| 规模等级分布 | `getScaleStats()`                         | 迷你柱状图/进度条 (S/A/B/C/D)      |
| 消费等级分布 | `getConsumeLevelStats()`                  | 迷你柱状图/进度条 (S/A/B/C/D)      |
| 房产客户占比 | `getRealEstateStats()`                    | 百分比显示                         |

#### 2.3.2 ECharts 可视化区域 (Charts)

3 个图表，复用 `analytics/Profile.vue` 的图表配置模式：

| 图表名称       | 类型   | 数据来源 API                     | 维度               |
| -------------- | ------ | -------------------------------- | ------------------ |
| 行业分布       | 饼图   | `getIndustryDistribution()`      | 行业 + 客户数      |
| 规模等级分布   | 柱状图 | `getScaleStats()`                | 等级 (S/A/B/C/D)   |
| 消费等级分布   | 柱状图 | `getConsumeLevelStats()`         | 等级 (S/A/B/C/D)   |

#### 2.3.3 快捷操作区域 (QuickActions)

3 个快捷链接卡片：

| 卡片名称     | 跳转路由              | 图标     |
| ------------ | --------------------- | -------- |
| 标签管理     | `/tags`               | 🏷️       |
| 画像分析详情 | `/analytics/profile`  | 📊       |
| 客户列表     | `/customers`          | 👥       |

## 3. 技术实现

### 3.1 数据流

```
Dashboard.vue (mounted)
    │
    ├─ loadStats() ──→ Promise.allSettled([
    │                    getScaleStats(),
    │                    getConsumeLevelStats(),
    │                    getIndustryDistribution(),
    │                    getRealEstateStats()
    │                  ])
    │
    ├─ 解析结果 ──→ 更新响应式数据
    │
    └─ 渲染组件 ──→ StatsCards + Charts + QuickActions
```

### 3.2 错误处理

- 使用 `Promise.allSettled` 替代 `Promise.all`，部分 API 失败不影响其他模块
- Loading 状态：`a-spin` 包裹整个内容区域
- Empty 状态：所有数据为空时显示引导提示（"暂无画像数据，请先为客户创建画像"）
- 错误提示：`Message.error()` 显示失败模块信息

### 3.3 权限控制

- 页面级：路由 meta.permissions = `['profiles:view']`
- 组件级：快捷操作卡片根据 `tags:view`、`analytics:view`、`customers:view` 权限动态显示
