# 细粒度权限设计文档

**日期**: 2026-04-14
**状态**: 已确认
**作者**: AI Assistant

---

## 背景

当前系统权限粒度过粗，所有权限均为模块级（如 `customers:manage`），无法区分查看/编辑/删除等操作。拥有模块权限的用户自动获得该模块下所有操作能力，不符合最小权限原则。

## 设计目标

1. 将权限从模块级细分为操作级
2. 高风险操作（删除、导出、充值等）独立控制
3. 权限完全独立，无隐式继承关系
4. 权限码格式统一为 `{module}:{action}`

## 权限继承

**无隐式继承**。每个权限完全独立，管理员需要显式勾选每个需要的权限。

## 影响范围

### 后端

- `backend/scripts/seed.py` — 替换 `ALL_PERMISSIONS` 列表
- `backend/app/routes/*.py` — 更新路由装饰器中的权限校验
- `backend/app/middleware/auth.py` — 权限校验逻辑（无需改动，仅更新调用方）
- `backend/tests/` — 更新测试中的权限 seed 数据

### 前端

- `frontend/src/views/Dashboard.vue` — 更新侧边栏权限校验
- `frontend/src/views/**/*.vue` — 更新页面内按钮级权限控制
- `frontend/src/api/` — API 层权限校验（如需要）

## 迁移策略

1. 新增细粒度权限，保留旧权限向后兼容
2. 超级管理员角色自动获得所有新权限
3. 已有自定义角色保持不变，管理员手动调整
4. 旧权限码（如 `customers:manage`）标记为 deprecated，后续版本移除
