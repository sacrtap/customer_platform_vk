# 设计文档："我的"用户中心功能

**创建日期**: 2026-05-13
**状态**: 待批准
**作者**: AI Assistant

---

## 1. 需求概述

在系统顶部 Header 增加"我的"用户入口，提供个人信息查看/编辑、密码修改、退出登录等功能。消息中心保持独立按钮，位于"我的"左侧。

## 2. 交互设计

### 2.1 Header 布局（从左到右）

```
[搜索] [消息通知🔔] [系统设置⚙️] | [👤头像] [退出图标]
```

- **搜索**: 保持现有按钮
- **消息通知**: 铃铛图标 + 红色 badge，独立按钮
- **系统设置**: 齿轮图标，独立按钮
- **分隔线**: 视觉分隔
- **我的**: 圆形头像（40px），显示用户名首字母，渐变蓝色背景
- **退出登录**: 缩小为小图标（32px），放在头像右侧

### 2.2 "我的"下拉菜单

点击头像弹出传统下拉菜单（240px 宽，圆角 12px）：

```
┌─────────────────────────┐
│ [头像] admin             │  ← 用户信息头部（灰色背景）
│        超级管理员         │
├─────────────────────────┤
│ 👤 个人信息              │  ← 跳转到 /profile
│ 🔒 修改密码              │  ← 弹出密码修改对话框
├─────────────────────────┤
│ 🚪 退出登录              │  ← 红色文字
└─────────────────────────┘
```

**交互细节**:
- 点击头像打开/关闭菜单
- 点击菜单外部区域关闭菜单
- hover 菜单项时高亮背景
- 阴影: `0 8px 32px rgba(0,0,0,0.12)`

### 2.3 消息中心

保持现有独立按钮（铃铛图标），位置在"我的"头像左侧。

## 3. 个人信息页面设计

### 3.1 路由

- 路径: `/profile`
- 组件: `frontend/src/views/Profile.vue`

### 3.2 页面布局

```
┌─────────────────────────────────────┐
│  个人信息                            │
├─────────────────────────────────────┤
│                                     │
│  [圆形头像 80px]  [更换头像]         │
│                                     │
│  用户名      [admin] (只读)          │
│  邮箱        [admin@example.com]     │
│  手机号      [138****1234]           │
│  最后登录    2026-05-13 10:30        │
│                                     │
│              [保存] [取消]           │
│                                     │
└─────────────────────────────────────┘
```

### 3.3 字段说明

| 字段       | 类型     | 可编辑 | 说明                     |
| ---------- | -------- | ------ | ------------------------ |
| 用户名     | string   | 否     | 只读展示                 |
| 邮箱       | string   | 是     | 需邮箱格式验证           |
| 手机号     | string   | 是     | 需手机号格式验证         |
| 头像       | URL/file | 是     | 支持上传或输入 URL       |
| 最后登录时间 | string | 否   | 只读展示                 |

### 3.4 密码修改

- 交互方式: 对话框弹窗（复用现有忘记密码对话框样式）
- 字段: 当前密码、新密码、确认密码
- 验证: 新密码长度 ≥ 6，两次输入一致

## 4. 后端 API 设计

### 4.1 新增端点

| 方法   | 路径                    | 说明           | 认证 |
| ------ | ----------------------- | -------------- | ---- |
| GET    | `/api/users/profile`    | 获取个人信息   | 需要 |
| PUT    | `/api/users/profile`    | 更新个人信息   | 需要 |
| PUT    | `/api/users/password`   | 修改密码       | 需要 |
| POST   | `/api/users/avatar`     | 上传头像       | 需要 |

### 4.2 请求/响应示例

**GET /api/users/profile**
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "username": "admin",
    "email": "admin@example.com",
    "phone": "13812341234",
    "avatar_url": "https://...",
    "last_login_at": "2026-05-13T10:30:00Z"
  }
}
```

**PUT /api/users/profile**
```json
{
  "email": "new@example.com",
  "phone": "13912341234",
  "avatar_url": "https://..."
}
```

**PUT /api/users/password**
```json
{
  "current_password": "old123",
  "new_password": "new456"
}
```

### 4.3 数据库变更

需要在 `users` 表新增字段：
- `email` VARCHAR(255) NULL
- `phone` VARCHAR(20) NULL
- `avatar_url` VARCHAR(500) NULL
- `last_login_at` TIMESTAMP NULL

## 6. 技术实现要点

### 6.1 下拉菜单实现

使用 Vue 3 的 `ref` + `@click.outside` 实现点击外部关闭：
```typescript
const dropdownOpen = ref(false)
const toggleDropdown = () => { dropdownOpen.value = !dropdownOpen.value }
// 点击外部关闭：使用 Arco Design 的 a-popover 或自定义指令
```

推荐使用 Arco Design 的 `a-popover` 组件，内置点击外部关闭、定位等功能。

### 6.2 头像显示

- 默认：用户名首字母 + 渐变背景
- 有头像 URL 时：显示 `<img>` 标签
- 头像上传：支持文件选择 + 预览

### 6.3 权限约束

- 个人信息页面：仅当前登录用户可访问和编辑自己的信息
- 密码修改：需验证当前密码正确性

## 8. 风险和注意事项

1. **头像存储**: 需要确定头像存储方案（本地文件服务器 or OSS），本设计暂使用 URL 方式
2. **手机号脱敏**: 列表展示时脱敏（138****1234），编辑页显示完整号码
3. **邮箱唯一性**: 更新邮箱时需检查是否已被其他用户使用
4. **密码强度**: 建议增加密码强度验证（长度、复杂度）
