# 客户页面功能测试计划

**文档版本**: 1.0  
**创建日期**: 2026-04-09  
**测试范围**: 客户管理模块（Index.vue + Detail.vue）  
**参考文档**: [系统设计文档](../superpowers/specs/2026-04-01-customer-platform-design.md)

---

## 一、测试范围

### 1.1 页面覆盖

| 页面 | 文件路径 | 功能模块 |
|------|----------|----------|
| 客户列表页 | `frontend/src/views/customers/Index.vue` | 客户列表、筛选、CRUD、导入/导出 |
| 客户详情页 | `frontend/src/views/customers/Detail.vue` | 基础信息、画像信息、余额信息、结算单、用量数据、标签管理 |

### 1.2 功能模块

```
客户管理
├── 客户列表页 (Index.vue)
│   ├── 筛选功能
│   │   ├── 基础筛选（关键词、账号类型、业务类型、客户等级、重点客户）
│   │   └── 高级筛选（运营经理、标签组合）
│   ├── 客户 CRUD
│   │   ├── 新建客户
│   │   ├── 编辑客户
│   │   ├── 删除客户
│   │   └── 查看客户详情
│   ├── 批量操作
│   │   ├── Excel 导入客户
│   │   └── Excel 导出客户
│   └── 列表展示
│       ├── 表格渲染
│       └── 分页功能
│
└── 客户详情页 (Detail.vue)
    ├── 基础信息 Tab
    │   ├── 客户信息展示
    │   └── 客户标签管理
    ├── 画像信息 Tab
    │   └── 画像数据展示
    ├── 余额信息 Tab
    │   └── 余额卡片展示
    ├── 结算单 Tab
    │   └── 结算单列表
    ├── 用量数据 Tab
    │   └── 用量列表与分页
    └── 页面操作
        ├── 编辑客户
        └── 切换重点客户
```

## 二、测试环境

### 2.1 环境配置

| 项目 | 配置 |
|------|------|
| **前端框架** | Vue 3.4 + TypeScript 5.3 |
| **UI 组件库** | Arco Design 2.54 |
| **测试工具** | Playwright (E2E) |
| **浏览器** | Chrome 120+, Firefox 121+, Safari 17+ |
| **分辨率** | 1920x1080 (桌面), 375x667 (移动) |

### 2.2 测试数据

| 数据类型 | 数量 | 说明 |
|----------|------|------|
| 客户数据 | ≥50 条 | 包含各种类型、等级、业务类型 |
| 标签数据 | ≥20 个 | 客户标签分类 |
| 用户数据 | ≥5 个 | 运营经理账号 |
| 结算单数据 | ≥30 条 | 各种状态的结算单 |
| 用量数据 | ≥100 条 | 每日用量记录 |

## 三、测试策略

### 3.1 测试类型

| 类型 | 比例 | 说明 |
|------|------|------|
| 功能测试 | 60% | 验证业务功能正确性 |
| UI/视觉测试 | 15% | 验证界面与设计一致性 |
| 交互测试 | 15% | 验证用户交互体验 |
| 边界测试 | 10% | 验证边界条件和异常处理 |

### 3.2 测试优先级

| 优先级 | 功能模块 | 测试用例数 |
|--------|----------|------------|
| P0 (Critical) | 客户 CRUD、登录认证 | 15 |
| P1 (High) | 筛选功能、导入/导出、标签管理 | 25 |
| P2 (Medium) | 详情页数据展示、分页 | 15 |
| P3 (Low) | 视觉细节、响应式 | 10 |

#### TC-INDEX-002: 新建客户 - 成功
**优先级**: P0  
**类型**: 功能测试

**前置条件**:

**测试步骤**:
1. 点击"新建客户"按钮

4. 点击"确定"

**预期结果**:

- 对话框关闭
- 列表刷新显示新客户
- 客户数据正确保存

**测试数据**:
```json
{
  "company_id": "TEST001",
  "name": "测试客户",
  "email": "test@example.com",
  "account_type": "正式账号",
  "business_type": "A",
  "customer_level": "KA",
```

## 六、Bug 管理

### 6.1 Bug 分级

| 级别 | 定义 | 示例 |
|------|------|------|
| Critical (P0) | 系统崩溃、数据丢失、核心功能不可用 | 客户无法创建、列表无法加载 |
| High (P1) | 主要功能异常、无替代方案 | 筛选功能失效、导入失败 |
| Medium (P2) | 部分功能异常、有替代方案 | 导出格式错误、标签无法移除 |
| Low (P3) | 视觉问题、边缘场景 | 按钮对齐偏差、提示文字错误 |
