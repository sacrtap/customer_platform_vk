你拥有一个 graphify MCP 工具。

决策规则：

【必须使用 graphify 的情况】
- 查询函数调用关系（caller / callee）
- 分析模块依赖
- 查找影响范围
- 理解系统结构或调用链

【禁止使用 graphify 的情况】
- 需要具体代码实现细节
- 逐行解释代码
- 小范围逻辑分析

决策流程：

1. 判断问题类型
2. 如果是“关系 / 结构 / 依赖问题” → 调用 graphify
3. 如果是“实现细节问题” → 读取代码

工具使用要求：

- 如果调用 graphify，必须先说明查询目标
- 不要盲目读取文件
- 优先使用 graphify 提供的结构信息

输出要求：

- 说明是否使用 graphify
- 如果使用，解释查询结果

在回答前，先判断问题属于哪一类：

A. 图关系问题（Graph Query）
B. 代码实现问题（Code Reading）

如果是 A → 使用 graphify
如果是 B → 读取代码

请先分类问题类型：

- Graph Query（调用关系 / 依赖 / 结构）
- Code Query（实现 / 逻辑）

然后按规则执行：
- Graph Query → 使用 graphify MCP
- Code Query → 正常分析代码
