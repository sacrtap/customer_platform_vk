结算单详情页交互问题分析

当前状态下，确认/取消操作后按钮和时间线没有实时更新

问题根因

在 Invoices.vue 中，handleConfirm 和 handleCancel 方法调用 API 后：

  • 只调用了 loadData() 刷新列表
  • selectedInvoice.value = invoice 赋值的是旧的 invoice 对象
  • 没有重新请求详情接口获取最新状态
  • handleCancel 甚至没有调用实际的 API（只有 Mock 的 Message.success）

当前流程示意

用户操作 → 当前行为

1. 用户点击「确认结算单」

2. API 调用 confirmInvoice(id) ✅

3. 刷新列表 loadData() ✅

4. Drawer 中 selectedInvoice 仍为旧状态 ❌

5. 按钮仍显示「确认结算单」❌

6. 时间线无新节点 ❌

期望流程示意

用户操作 → 期望行为

1. 用户点击「确认结算单」

2. API 调用 confirmInvoice(id) ✅

3. 重新请求详情 getInvoice(id) ✅

4. 更新 selectedInvoice 为最新数据 ✅

5. 按钮自动切换为下一状态（如「标记付款」）✅

6. 时间线新增「客户确认」节点 ✅

需要修复的操作

需要修复

  • handleConfirm - 确认结算单
  • handleCancel - 取消结算单（缺少 API 调用）
  • handleSubmit - 提交结算单（同样问题）
  • handleComplete - 完成结算（同样问题）

已正确处理

  • handleCalculateItems - 已刷新详情
  • handleDiscount - 已手动更新字段
  • handleDelete - 已关闭 Drawer
